# Getting Started with Markdown: A Beginner's Guide

---

## Introduction

If you've ever wanted to write for the web without wrestling with HTML, **Markdown** is your new best friend. It's a lightweight formatting language that converts plain text into beautifully structured documents — and once you learn it, you'll wonder how you ever lived without it.

> "Markdown is intended to be as easy-to-read and easy-to-write as is feasible." — John Gruber

---

## Why Markdown?

Here's why so many writers and developers have switched to Markdown:

- **Simple syntax** — learn the basics in under 10 minutes
- **Portable** — just a plain `.md` text file, works everywhere
- **Readable raw** — even without rendering, it looks clean
- **Widely supported** — GitHub, Notion, VS Code, and hundreds of other tools

---

## The Essentials

### Headings

Use `#` symbols to create headings. One `#` is a big heading, six `######` is the smallest.

### Emphasis

You can make text **bold** with double asterisks, or *italic* with single asterisks. You can even do ***both at once***.

### Lists

Unordered lists use dashes or asterisks:

- Item one
- Item two
  - Nested item
  - Another nested item

Ordered lists are just numbers:

1. First step
2. Second step
3. Third step

### Code

Inline code uses backticks: `print("Hello, world!")`

For larger blocks, use triple backticks:

```python
def greet(name):
    return f"Hello, {name}!"

print(greet("World"))
```
